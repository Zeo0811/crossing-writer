# v1.6 · UI 库统一 — Design Spec

- **Spec ID**: v1.6-ui-library
- **Date**: 2026-04-15
- **Status**: Approved
- **Predecessors**: v1.5.0 UI 改版（已发布）
- **Successors**: v1.7（交互升级 + 密度调整）· v1.8（视觉调性） → v2.0

---

## 1. 动机

v1.5 跑通后 80% 视觉对齐 mock，但剩 20% 依靠手工散落样式支撑：

- `<button className="px-3 py-1 rounded bg-accent …">` 在 15+ 处重复，每次改都要到处 grep
- `<input>` / `<select>` / `<textarea>` 的样式靠 copy paste，`bg-bg-2 border border-hair rounded px-3 py-2 text-sm outline-none focus:border-accent-soft` 随处可见
- Modal / Dropdown / Tabs 各自为政，a11y 支持参差（有的不能 Esc 关闭，有的 Tab 陷入）
- `text-base / text-lg / text-sm` 层级在不同页混用
- `bg-white / text-red-600` 等老 tailwind 色还散在没替换完

继续手工维护，改 token 一次要碰 40+ 文件，交付节奏会被拖死。

v1.6 一次性把所有散落 UI 收敛到统一组件库，后续改 token / 调布局只改一处。

## 2. 锁定决策

| 维度 | 选择 |
|---|---|
| 路线 | 项目内自建 UI 库 |
| 依赖 | 自建 + Radix headless（复杂交互用 Radix） |
| 落地节奏 | v1.6（本 spec）→ v1.7 → v1.8 |
| 外部库 | 仅 Radix、clsx、tailwind-merge、CVA（class-variance-authority） |
| 保留 | 现有 `tailwind @theme` token / dark-light 两套主题不动 |

## 3. 组件清单（18 个）

### 3.1 基础

| 组件 | API 核心 |
|---|---|
| `Button` | `variant=primary/secondary/ghost/danger/link`，`size=sm/md/lg`，`loading`，`disabled` |
| `Card` | `variant=outer/nested/ghost`，`padding=sm/md/lg` |
| `Chip` | `variant=neutral/accent/amber/red/pink`，`tone=solid/soft`，`size=sm/md` |
| `StatusBadge` | 从 `statusBadge(status)` 返回值驱动，封装 chip |

### 3.2 表单

| 组件 | API 核心 |
|---|---|
| `Input` | error 状态，leftSlot/rightSlot |
| `Textarea` | rows，autoResize 可选 |
| `Select` | Radix Select 封装，support group |
| `Checkbox` | Radix Checkbox |
| `Radio` | Radix RadioGroup |
| `FormField` | `label` + `hint` + `error` 外壳，包裹任意 children |

### 3.3 结构 / 反馈

| 组件 | API 核心 |
|---|---|
| `Tabs` | Radix Tabs，默认 underline 激活样式 |
| `Dialog` | Radix Dialog（含 backdrop-blur + bg-bg-1 panel） |
| `Menu` | Radix DropdownMenu |
| `Tooltip` | Radix Tooltip |
| `Toast` | 迁移自 ToastProvider，API 规范 `toast.success/error/info` |
| `EmptyState` | icon slot + title + body + primary CTA |

### 3.4 业务壳

| 组件 | API 核心 |
|---|---|
| `SectionTitle` | `level=h1/h2/h3`，配 `action` 右上操作位 |
| `PhaseSteps` | 搬迁自 `components/layout/PhaseSteps`，API 不变 |

## 4. 类型层级标准

| 层级 | 用途 | 样式 |
|---|---|---|
| `h1` | 页面大标题 | `text-lg font-semibold text-heading`（18px） |
| `h2` | section / card 内主标题 | `text-base font-semibold text-heading`（16px） |
| `h3` | 组内小标题 | `text-sm font-semibold text-meta`（14px） |
| body | 正文 | `text-sm text-body` |
| meta | field label / 辅助信息 | `text-xs text-meta` |
| faint | 时间戳 / 计数 | `text-[10px]` 或 `text-xs` `text-faint` |

像素字（`font-pixel`）**仅**用于：Logo `CROSSING.WRITER` + 版本 chip `v1.5` + 少量 PhaseSteps 序号（如确定用）。

## 5. 色彩与间距标准

### 5.1 色彩

- 组件层 **禁止** 硬编码 hex、禁止用 `text-red-600 / bg-white / border-gray-300` 等 tailwind 默认色
- 只使用 CSS var 或对应的 `@theme` tailwind token
- 语义色：`accent` / `amber` / `red` / `pink` 四色 + `fg` / `bg` / `hair` 三层

### 5.2 间距

| 场景 | 值 |
|---|---|
| 页面容器 | `max-w-[1280px] mx-auto px-5 pt-7 pb-[72px] flex-col gap-7` |
| 页内卡外壳 | `rounded border border-hair bg-bg-1 overflow-hidden` |
| 页头 header | `px-6 h-14`（从原 h-12 升） |
| 主区 padding | `px-6 py-5` |
| 嵌套面板 | `bg-bg-2 rounded p-4` 或 `p-[18px]` |
| 行间距 | `gap-3`（紧）/ `gap-4`（默认）/ `gap-5`（疏） |

## 7. 迁移策略

### Step 1：建库
- 一次写完 18 个组件 + barrel + cn + variants
- 每个组件 100-300 行，含 variants 与 a11y 属性
- 不写单测，走 tsc + vite build 保证编译通过

### Step 2：全站替换（grep 驱动）
- `<button className="..."`  → `<Button variant="...">`
- `<input className="..."` → `<Input />`
- `<select className="..."` → `<Select>`
- modal 手写层 → `<Dialog>`
- 硬编码 `bg-white/text-red-600` → 对应 token
- 每轮提交保持测试绿 + tsc 干净

### Step 3：清理
- 删老的散装组件（ui/Button.tsx 等旧版）
- 补 `components/ui/index.ts` 作为唯一 import 源
- 删老的 snapshot / 过期测试

### Step 4：Review
- 手工点每一页验收
- 对照 mock `?mock=1` 确认视觉不倒退

## 9. 不做（v1.6 out of scope）

- 任何动画系统（留给 v1.7）
- Table / DataTable / Pagination
- DatePicker / TimePicker
- 键盘快捷键扩展（留给 v1.7）
- 视觉 token / 色值调整（留给 v1.8）
- Command Palette 抽出为组件（仍在 mock/，实际产品未启用）

## 10. 验收标准

- 所有 `.tsx` 文件 grep `bg-white|text-red-\d|bg-gray|rounded-\[6px\]` 必须零匹配
- 所有 `<button>` 换成 `<Button>`（UI 库之外只允许 icon-only small buttons 使用原生 `button`）
- 所有 modal / dialog 走 `<Dialog>`
- `pnpm exec tsc --noEmit` 干净（不含 pre-existing 错误）
- `pnpm build` 通过
- 视觉对照 mock 页面（`?mock=1`）不倒退
- 手工测试：ProjectList / Workbench 各 phase / Knowledge / Style / Config / Settings 全部能正常渲染

## 11. 风险

- Radix 依赖体积：+40KB gzipped，值得（a11y 自带）
- 手工替换量大：预估 40-60 个文件被改。分批提交降低 review 成本
- 老 `@theme` tailwind utility（`text-heading / bg-bg-1`）在 components/ui 外仍然使用，v1.6 不强行改，v1.8 统一清理
